01 Суть продукта

Зачем это нужно рынку

Бухгалтер выгружает XML-декларацию НДС из 1С, загружает в NalogAI, и через 2 секунды получает полный аудит: какие ошибки найдены, какие суммы под угрозой, и что конкретно исправить в 1С, чтобы избежать камеральной проверки.

Каждый квартал ~3,5 млн организаций сдают декларацию НДС. ФНС использует АСК НДС-2 — автоматическую систему, которая проверяет контрольные соотношения и отбирает декларации с высокой долей вычетов для углублённой камеральной проверки. NalogAI делает ту же проверку, но ДО подачи.

1 XML из 1С Выгрузка декларации
→
2 Парсинг 9 разделов, 24+ поля
→
3 Check Engine 22 проверки, 2 слоя
→
4 Отчёт Ошибки + fix в 1С

03 Безопасные доли вычетов по регионам

85 регионов РФ. Источник: Отчёт ФНС 1-НДС за Q3 2025. Обновляется ежеквартально.

Раздел 3 — Расчёт налога (24 поля)

База и НДС по ставкам 20%, 18%, 10%, 0%. Авансы полученные (стр.070), восстановленный НДС (стр.080). Итого начислено (стр.118). Вычеты по товарам (стр.120), авансам (стр.130), ввозу ЕАЭС (стр.150), таможне (стр.160). Итого вычеты (стр.190). К уплате (стр.200) / к возмещению (стр.210).

Разделы 8–9 — Книги покупок/продаж

Каждый счёт-фактура: ИНН/КПП контрагента, номер и дата СФ, сумма с НДС (копейки), сумма НДС (копейки), ставка налога, признак корректировки. Итоги по книгам. Типичный файл: 5–500 СФ.

Алгоритм валидации ИНН

Встроена проверка контрольных цифр для ИНН юридических лиц (10 знаков) и физических лиц (12 знаков). Весовые коэффициенты: [2,4,10,3,5,9,4,6,8] для ЮЛ, [7,2,4,10,3,5,9,4,6,8] + [3,7,2,4,10,3,5,9,4,6,8] для ФЛ. Проверяется каждый ИНН контрагента из книги покупок — невалидный ИНН означает потенциальный отказ в вычете.

Совместимость парсера

Content-based detection: парсер не привязан к имени файла или конкретной версии XSD. Определяет тип XML по набору тегов внутри документа. Поддерживает windows-1251 и UTF-8 кодировки (обе используются 1С). Работает со всеми версиями деклараций НДС с 2019 года.

RAG-корпус (Фаза 2 — в разработке)

Для Layer 3 (AI-рекомендации) планируется индексация полного корпуса налогового законодательства:

05 Архитектура и стек

Минимальный стек для MVP, масштабируемый до production

API

Backend

FastAPI + SQLAlchemy + SQLite
Синхронная обработка файлов (XML < 5 МБ), REST API с 4 эндпоинтами. В production — PostgreSQL + pgvector.

UI

Frontend

Next.js 15 + App Router + Tailwind
Drag-and-drop загрузка XML/ZIP. Дашборд результатов с карточками проблем. shadcn/ui компоненты.

XML

XML-парсер

Content-based type detection
Не привязан к именам файлов или конкретной XSD-версии. Работает со всеми версиями XML декларации НДС с 2019 года. Поддержка windows-1251 и UTF-8.

AI

AI (Фаза 2)

Claude Sonnet 4.6 + Voyage-3 + pgvector
RAG по Налоговому кодексу и письмам ФНС. Персонализированные рекомендации с ссылками на конкретные статьи.

09 Ключевые архитектурные решения

Почему так, а не иначе

Копейки (int) вместо рублей (float)

Все суммы хранятся в копейках как целые числа. Это исключает ошибки IEEE 754 при арифметических проверках. Пример: 0.1 + 0.2 ≠ 0.3 в float, но 10 + 20 = 30 в int. Для декларации НДС это критично — расхождение в 1 копейку может дать ложный RED.

Content-based detection вместо привязки к XSD

ФНС меняет XSD-схемы каждый год (v5.09, v5.10, v5.11...). Парсер ищет разделы по набору тегов в XML, а не по структуре конкретной версии. Это позволяет работать со всеми версиями деклараций с 2019 года без обновления кода при смене формата.

Регион по ИНН, а не по ОКТМО

ОКТМО для Москвы начинается с "45", но безопасные доли вычетов ФНС публикует по коду региона "77". Использование первых 2 цифр ИНН надёжнее — они всегда совпадают с кодом региона регистрации организации.

SQLite для MVP, PostgreSQL + pgvector для production

SQLite не требует отдельного процесса и деплоя. Для MVP с одним пользователем — идеально. При переходе к production: PostgreSQL + pgvector (для RAG embeddings) на Selectel VPS.

Selectel VPS, а не Vercel / AWS

152-ФЗ: персональные данные (ИНН, КПП, наименования организаций) должны храниться на территории РФ. Vercel и AWS не имеют ЦОД в России. Selectel: рублёвые платежи, SLA 99.9%, 152-ФЗ compliance из коробки.

Синхронная обработка без Celery

XML-декларация НДС — это файл 0.5–5 МБ. Парсинг + проверка занимает < 500 мс. Celery, Redis, worker — оверинжиниринг для такого объёма. Если файл > 10 МБ — API возвращает 413.
